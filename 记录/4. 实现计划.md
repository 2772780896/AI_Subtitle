# 实现计划

## 实现策略

**先通后完善**：先让前后端能"对话"（返回 mock 数据），再逐步把 mock 换成真逻辑。

## 四层实现顺序

### 第一层：后端骨架（能跑起来）

**目标：** Express 服务启动，接口能返回 mock 数据

- [ ] 安装依赖：`npm install express dotenv cors multer uuid`
- [ ] 写 `backend/app.js`（Express 入口 + 中间件 + 路由注册）
- [ ] 写 `backend/routes/subtitle.js`（4 个路由 + mock 响应）
- [ ] 创建 `.env`（TENCENTCLOUD_SECRET_ID / SECRET_KEY）
- [ ] 创建目录：`backend/storage/input/` `backend/storage/temp/` `backend/storage/output/`
- [ ] 验证：`npm run dev` → 浏览器访问 `http://localhost:3000/api/status/test`

---

### 第二层：前端骨架（能交互）

**目标：** 前端能上传视频、配置样式、轮询状态、展示结果（用 mock 数据）

- [ ] 重写 `frontend/src/components/ASRPanel.jsx`
  - 视频文件上传（input type="file" accept="video/mp4"）
  - 字幕样式配置（字体、颜色、位置等）
  - ASR 配置（引擎选择、结果格式）
  - 上传按钮 → POST /api/upload（FormData）
  - 轮询状态 → GET /api/status/:taskId
  - 结果展示 + 下载链接
- [ ] 配置 Vite 代理（前端请求转发到后端 3000 端口）
- [ ] 验证：选视频 → 点上传 → 看到 mock 状态变化 → 看到 mock 结果

---

### 第三层：后端核心逻辑（逐个填入）

**目标：** 把 mock 响应替换为真实处理逻辑

按依赖顺序实现，每完成一个功能测试一次：

1. **文件上传 + 格式校验**（multer + MP4 检查）
   - 验证：上传 MP4 → 存到 storage/input/，上传其他格式 → 返回错误
   
2. **FFmpeg 提取音频**（`backend/services/ffmpegService.js`）
   - 验证：storage/input/xxx.mp4 → storage/temp/xxx.wav（16kHz/16bit/单声道）
   
3. **ASR 语音识别**（对接已有 `asrService.js`）
   - 验证：storage/temp/xxx.wav → 拿到 TaskId → 轮询拿到识别结果
   
4. **数据清洗 + SRT 生成**（`backend/services/srtGenerator.js`）
   - 验证：ASR 结果 → 清洗 → storage/output/xxx.srt + storage/output/xxx.raw.txt
   
5. **FFmpeg 烧录字幕**（扩展 `ffmpegService.js`）
   - 验证：storage/input/xxx.mp4 + storage/output/xxx.srt → storage/output/xxx.mp4

6. **日志系统**（`backend/services/logger.js`）
   - 写入 `storage/output/xxx.log` + console.log
