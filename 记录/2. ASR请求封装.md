# ASR 请求封装

## 数据格式

### 函数签名
```js
sendRequest(action, payload) → Promise<Object>
```

### 输入参数

| 参数 | 类型 | 说明 |
|------|------|------|
| action | String | 接口动作：`"CreateRecTask"` 或 `"DescribeTaskStatus"` |
| payload | Object | 请求体，会被自动 JSON.stringify |

#### CreateRecTask 常用 payload 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| EngineModelType | String | ✅ | 引擎：`16k_zh` 中文、`16k_zh_en` 普方英大模型、`16k_en` 英语、`16k_ja` 日语 |
| ChannelNum | Integer | ✅ | 声道数：`1` 单声道、`2` 双声道（仅8k电话音频） |
| ResTextFormat | Integer | ✅ | 结果格式：`0` 基础、`1` +词时间戳、`2` +标点、`3` 按标点分段（字幕用） |
| SourceType | Integer | ✅ | 数据来源：`0` URL、`1` base64音频数据 |
| Url | String | 条件 | 音频URL（SourceType=0时必填） |
| Data | String | 条件 | base64音频数据（SourceType=1时必填） |
| SpeakerDiarization | Integer | 否 | 说话人分离：`0` 关闭、`1` 开启 |

#### DescribeTaskStatus payload 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| TaskId | Integer | ✅ | 从 CreateRecTask 返回的任务ID |

### 返回数据

返回腾讯云原始响应对象（已 JSON.parse），结构：

```js
// CreateRecTask 成功响应
{
  Response: {
    RequestId: "xxx",
    Data: {
      TaskId: 1393265   // ← 用这个去轮询
    }
  }
}

// DescribeTaskStatus 成功响应
{
  Response: {
    RequestId: "xxx",
    Data: {
      TaskId: 1393265,
      Status: 2,           // 0=等待 1=执行中 2=成功 3=失败
      StatusStr: "success",
      AudioDuration: 2.38, // 音频时长（秒）
      Result: "[0:0.020,0:2.380] 识别文本...",
      ResultDetail: [      // 详细分段结果（ResTextFormat≥1时有）
        {
          FinalSentence: "腾讯云语音识别欢迎您。",
          StartMs: 20,
          EndMs: 2380,
          Words: [...]
        }
      ],
      ErrorMsg: ""
    }
  }
}

// 错误响应
{
  Response: {
    Error: {
      Code: "InvalidParameter",
      Message: "参数错误"
    },
    RequestId: "xxx"
  }
}
```

## 封装目标

把腾讯云官方的一次性脚本改造成可复用的函数，支持调用不同接口（CreateRecTask / DescribeTaskStatus）。

## 改造点

### 1. 参数化 action
原代码 action 写死为 "CreateRecTask"，改为函数参数，调用时传入：
```js
const sendRequest = (action, payload) => {
  // action 会设置到请求头 "X-TC-Action" 中
}
```

### 2. payload 处理
调用方传入对象，函数内部 JSON.stringify 转为字符串（签名和请求体都需要字符串）：
```js
payload = JSON.stringify(payload)
```
> 注意：不能用 `const payload = ...`，因为 payload 已是参数，同作用域不能重复声明。直接赋值即可。

### 3. 异步处理（核心）

**问题：** `https.request` 是回调式的，不是 Promise，不能直接 await。

**解决：** 用 `new Promise` 把回调部分包起来：
- `resolve(data)` — 在 `res.on("end")` 里调用，表示"数据回来了"
- `reject(error)` — 在 `req.on("error")` 里调用，表示"出错了"

**关键理解：**
- `async` 只能搭配 Promise，对回调无效，所以函数**不加 async**
- 函数本身返回 Promise，调用方用 `await` 等待结果

```js
const sendRequest = (action, payload) => {
  // ... 签名计算 ...

  return new Promise((resolve, reject) => {
    const req = https.request(options, (res) => {
      let data = ""
      res.on("data", (chunk) => { data += chunk })
      res.on("end", () => {
        resolve(JSON.parse(data))  // 成功，把结果交出去
      })
    })
    req.on("error", (error) => {
      reject(error)                // 失败，把错误交出去
    })
    req.write(payload)
    req.end()
  })
}
```

## 使用方式

```js
const { sendRequest } = require("./services/asrService")

// 提交识别任务
const result = await sendRequest("CreateRecTask", {
  EngineModelType: "16k_zh",
  ChannelNum: 1,
  ResTextFormat: 3,
  SourceType: 0,
  Url: "https://xxx/audio.wav"
})
const taskId = result.Response.Data.TaskId

// 轮询任务结果
const status = await sendRequest("DescribeTaskStatus", {
  TaskId: taskId
})
console.log(status.Response.Data.Status)  // 0等待 1执行中 2成功 3失败
```
