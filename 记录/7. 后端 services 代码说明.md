# 后端 services 代码说明

## 整体架构

```
route.js（HTTP 层，只负责收发）
  └─ processVideo.js（编排器，异步串联所有 service）
       ├─ ffmpegService.extractAudio()     提取音频
       ├─ asrService.sendRequest()         提交 ASR 识别
       ├─ asrService.sendRequest()         轮询 ASR 结果
       ├─ srtGenerator.cleanAndGenerate()  数据清洗 + 生成 SRT
       ├─ ffmpegService.burnSubtitle()     烧录字幕
       └─ 清理临时文件，设置下载 URL
```

route.js 调用 processVideo 后立即返回响应，前端通过轮询 `/api/status` 获取进度。

## 6 个文件各自职责

### store.js — 全局任务状态

- 一个 `Map` 对象，key 是 taskId，value 是任务对象
- route.js 和 processVideo.js 共享同一个 Map（独立文件，避免循环依赖）
- 任务对象包含公开字段（status/progress/videoUrl）和内部字段（_videoPath/_style/_asrConfig）

### logger.js — Winston 日志

- `appLogger`：应用级，只输出控制台
- `createTaskLogger(taskId)`：任务级，同时输出控制台和 `storage/output/{taskId}.log` 文件
- 每个任务独立一份日志文件，方便排查问题

### ffmpegService.js — FFmpeg 命令封装

**3 个核心函数：**

- `extractAudio(input, output)`：从视频提取单声道 16kHz WAV 音频
  - FFmpeg 参数：`-vn -acodec pcm_s16le -ar 16000 -ac 1`
  - 输出 PCM 格式是腾讯云 ASR 要求的标准格式

- `burnSubtitle(video, srt, output, style)`：将 SRT 字幕烧录到视频
  - 先调 `getVideoResolution()` 获取视频宽高
  - marginV 百分比 → 像素换算：`Math.round(高度 × marginV / 100)`
  - 颜色转换：前端 `#RRGGBB` → ASS `&H00BBGGRR`（BGR 反序）
  - SRT 路径用相对路径（避免 Windows 绝对路径在 FFmpeg 滤镜中解析出错）

- `getVideoResolution(videoPath)`：用 ffprobe 获取视频分辨率

**底层依赖：** `runCommand(cmd, args)` 用 `child_process.execFile` 执行命令，返回 Promise

### asrService.js — 腾讯云 ASR 请求

- `sendRequest(action, payload)`：通用腾讯云 API 请求函数
- 负责 TC3-HMAC-SHA256 签名算法、HTTP POST 请求、响应解析
- 被 processVideo 以两种 action 调用：
  - `CreateRecTask`：提交音频识别任务，返回 TaskId
  - `DescribeTaskStatus`：查询识别状态，Status=2 时返回结果

**关键：响应结构是 `Response.Data.TaskId` / `Response.Data.Status`，不是直接在 Response 下**

### srtGenerator.js — 数据清洗 + SRT 生成

**主入口：** `cleanAndGenerate(asrResult, baseName, log)`

接收 ASR 返回的原始识别数据，经过清洗后生成 SRT 文件。

**清洗流水线：** 去语气词 → 合并短句 → 拆长句 → 再合并一次 → 过滤空文本 → 限制时长 → 验证

**输出两个文件：**
- `{baseName}.srt`：清洗后的标准 SRT 字幕
- `{baseName}.raw.txt`：原始识别结果备份

> 详见 `5. srtGenerator 数据清洗与SRT生成.md`

### processVideo.js — 主编排器

**职责：** 接收 taskId，按顺序调用上面所有 service，每步更新任务状态。

**关键设计：**
- 整个函数一个 try/catch，任意步骤失败 → 更新 status='failed'
- ASR 请求带重试（最多 3 次，间隔 2 秒），防止偶发网络错误
- ASR 轮询：每 5 秒查一次，最多等 5 分钟（60 次）
- 轮询完成后将 `ResultDetail` 数组转换为 srtGenerator 需要的格式：
  - `FinalSentence` → `Text`
  - `StartMs / 1000` → `StartTime`（秒）
  - `EndMs / 1000` → `EndTime`（秒）

**状态流转和进度：**

| 步骤 | status | progress |
|------|--------|----------|
| 提取音频 | extracting | 15% |
| 提交 ASR | recognizing | 30% |
| ASR 完成 | recognizing | 60% |
| 数据清洗 | cleaning | 70% |
| SRT 生成 | generating | 80% |
| 烧录字幕 | rendering | 85% |
| 完成 | done | 100% |

## 文件间依赖关系

```
store.js  ←── route.js（读写任务状态）
store.js  ←── processVideo.js（读写任务状态）
processVideo.js → ffmpegService.js
processVideo.js → asrService.js
processVideo.js → srtGenerator.js
processVideo.js → logger.js（每个 service 也接收 log 参数）
```

## 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 循环依赖警告 | route.js ↔ processVideo.js 互相引用 | 抽出 store.js 独立存放 tasks Map |
| FFmpeg subtitles 路径解析失败 | Windows 绝对路径含驱动器号和中文 | SRT 路径改用相对路径 |
| ASR TaskId 为 undefined | 响应结构是 `Response.Data.TaskId` 不是 `Response.TaskId` | 修正为 `Response.Data.*` 取值 |
| 拆分长句产生过短尾部 | splitLong 末尾残余不足 1 秒 | mergeShort 再跑一次兜底 |
