# ASRPanel.jsx 代码说明

> 414 行，单组件，负责整个前端交互流程：上传 → 配置 → 预览 → 处理 → 下载。

## 1. 状态设计（6 个 useState）

| 状态 | 类型 | 职责 |
|------|------|------|
| `videoFile` | File | 用户选择的视频文件 |
| `blobUrl` | String | 由 videoFile 生成的临时预览 URL |
| `aspectRatio` | String | 视频容器宽高比，用于消除黑边 |
| `style` | Object | 字幕样式配置（字体/颜色/描边/位置） |
| `asrConfig` | Object | ASR 识别引擎配置 |
| `task` | Object | **任务生命周期**：id、status、progress、loading、errorMsg、三个下载 URL |

任务相关原本有 8 个独立 useState，合并为一个 `task` 对象后更清晰——这些字段总是联动更新。

另外 3 个 `useRef`：
- `fileInputRef`：重置时清空 input 的 value
- `videoContainerRef`：拖拽时获取容器高度做百分比换算
- `subtitleRef`：预留，当前未用

## 2. Blob URL 管理（L47-56）

```
videoFile 变化 → useEffect → createObjectURL → 设到 blobUrl
组件卸载或 videoFile 变化 → 清理函数 → revokeObjectURL 释放内存
```

- 不上传到服务器，纯浏览器本地预览
- `revokeObjectURL` 防止内存泄漏（每次换文件都会回收旧 URL）

## 3. 字幕拖拽（L58-79）

**三步：mousedown → mousemove → mouseup**

1. `mousedown`：记录起始 Y 坐标和当前 marginV 百分比
2. `mousemove`：算出鼠标位移（像素）→ 除以容器高度 → 转为百分比差值 → 更新 marginV
3. `mouseup`：移除 document 上的事件监听

**关键细节：**
- 容器高度通过 `videoContainerRef.current.clientHeight` 动态获取（不硬编码）
- `Math.max(0, Math.min(100, ...))` 限制范围在 0%~100%
- 事件挂在 `document` 上而非字幕元素上，防止鼠标移出元素后拖拽失效

## 4. 描边 CSS（L81-90）

CSS 没有 `-webkit-text-stroke` 的描边+填充色控制，所以用 **4 个方向的 text-shadow** 模拟 ASS 字幕的 Outline 效果：

```
text-shadow: -2px -2px 0 #000, 2px -2px 0 #000, -2px 2px 0 #000, 2px 2px 0 #000
```

outline=0 时返回 `none`，不渲染描边。

## 5. 上传流程（L92-118）

```
FormData 打包三个字段 → POST /api/upload → 拿到 taskId → 触发轮询
```

- `video`：File 对象（浏览器自动设置 multipart boundary）
- `style`：`JSON.stringify(style)` 序列化为字符串
- `asrConfig`：同上
- 用 `res.ok` 判断 HTTP 状态（fetch 不会自动抛异常）

## 6. 轮询状态（L120-145）

```
task.id 有值 → useEffect 启动 setInterval → 每 2 秒 GET /api/status/:id
```

- `done`：存储三个下载 URL，清除定时器
- `failed`：存储错误信息，清除定时器
- **清理函数** `return () => clearInterval(timer)`：组件卸载或 task.id 变化时自动清除，防止内存泄漏和重复请求

## 7. 视频预览消除黑边（L185-200）

问题：容器默认 16:9，但视频可能是 4:3 或 9:16，不匹配时出现黑边。

解决：
- 默认 `aspectRatio = '16/9'`
- 视频加载触发 `onLoadedMetadata` → 读取 `videoWidth / videoHeight` → 更新 aspectRatio
- video 元素用 `object-cover` 填满容器，容器 `overflow-hidden` 裁切溢出

## 8. UI 区块（L159-411）

共 7 个 section，按流程顺序排列：

| 区块 | 渲染条件 | 内容 |
|------|----------|------|
| 视频上传 | 始终 | file input + 文件名/大小 |
| 字幕预览 | `blobUrl` 存在 | 视频播放器 + 可拖拽字幕层 |
| 字幕样式 | 始终 | 6 个配置项（字体/字号/颜色/描边/距边缘） |
| ASR 配置 | 始终 | 识别引擎 + 结果格式 |
| 操作按钮 | 始终 | 开始处理 / 重新开始 |
| 进度展示 | `task.status` 有值 | 进度条 + 状态文字 |
| 结果下载 | `task.status === 'done'` | 视频/SRT/原始识别 三个下载按钮 |

## 9. 技术选型理由

| 选择 | 理由 |
|------|------|
| fetch 而非 axios | 项目简单，无需额外依赖；fetch 是浏览器原生 API |
| Tailwind CSS 而非 Ant Design | 需求无复杂组件，Tailwind 更轻量 |
| 单组件不拆分 | 单页面单功能，状态高度耦合，拆分反而增加复杂度 |
| task 对象而非 useReducer | 语法直观，`useState` 足够，不引入新 API |
| 字幕位置用百分比 | 适配不同分辨率视频，后端按视频高度×百分比换算像素 |
| Vite Proxy 而非直连端口 | 避免 CORS 跨域问题，开发环境统一走 /api 前缀 |
