# AI 字幕系统 - 完整流程设计

## 用户操作流程

```
┌─────────────────────────────────────────────────────────────────┐
│  第一步：上传视频                                                 │
│  ├── 选择视频文件（MP4，30秒~3分钟）                              │
│  └── 前端显示文件名、大小，校验格式                               │
├─────────────────────────────────────────────────────────────────┤
│  第二步：配置字幕样式（与上传同时进行）                            │
│  ├── 字体选择：微软雅黑 / 思源黑体 / 其他                        │
│  ├── 字体大小：滑块或数字输入（16~36）                            │
│  ├── 字体颜色：颜色选择器（默认白色）                             │
│  ├── 描边颜色：颜色选择器（默认黑色）                             │
│  ├── 描边粗细：0~5（默认2）                                      │
│  ├── 字幕位置：顶部 / 居中 / 底部（默认底部）                     │
│  ├── 距边缘距离：滑块（0~100%，默认10）                        │
│  └── 实时预览：在示例图上展示效果                                 │
├─────────────────────────────────────────────────────────────────┤
│  第三步：点击"开始处理"                                          │
│  └── 前端上传视频 + 样式参数 → 后端                              │
├─────────────────────────────────────────────────────────────────┤
│  第四步：等待处理（前端轮询状态）                                  │
│  ├── 上传中...                                                   │
│  ├── 提取音频中...                                               │
│  ├── 语音识别中...                                               │
│  ├── 生成字幕中...                                               │
│  └── 渲染视频中...                                               │
├─────────────────────────────────────────────────────────────────┤
│  第五步：查看结果                                                 │
│  ├── 预览带字幕的视频（在线播放）                                 │
│  ├── 下载字幕文件（.srt）                                        │
│  └── 下载带字幕的视频（.mp4）                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 后端处理流程

> **架构：** route.js 只负责接收请求和返回响应，实际处理逻辑由 `processVideo.js` 编排，按顺序调用其他 service。
> route.js 调用 processVideo 后立即返回响应（fire-and-forget），不阻塞 HTTP 请求。

> **注意：**
> - 所有异步操作需加 try/catch，失败时更新任务状态为 `failed` 并记录 errorMsg
> - ASR API 调用需支持自动重试（最多 3 次，间隔 2 秒）
> - 日志使用 Winston（`logger.js`），每个 service 的入口和出口各记一条，加任何 catch 到的错误
> - 日志同时输出到控制台和文件 `storage/output/xxx.log`

### processVideo 编排器

```
route.js（返回响应给前端）
  └─ processVideo.js（异步执行）
       ├─ 1. ffmpegService.extractAudio()    → task.status = 'extracting'
       ├─ 2. asrService.sendRequest()         → task.status = 'recognizing'
       ├─ 3. asrService.pollResult()          → 轮询直到拿到结果
       ├─ 4. srtGenerator.cleanAndGenerate()  → task.status = 'cleaning' + 'generating'
       ├─ 5. ffmpegService.burnSubtitle()     → task.status = 'rendering'
       └─ 6. 清理临时文件                      → task.status = 'done'
```

每步之间更新 `tasks.get(taskId)` 的 status 和 progress，前端轮询自动拿到最新状态。

### 处理步骤

```
接收请求
  │
  ▼
1. 保存视频 → storage/input/xxx.mp4
  │
  ▼
2. FFmpeg 提取音频
   命令：ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 temp/xxx.wav
  （注：实际路径为 storage/input/ 和 storage/temp/）
  │
  ▼
3. 调用腾讯云 ASR（CreateRecTask）
   参数：用户配置的 engineType / resTextFormat
   返回：TaskId
  │
  ▼
4. 轮询 ASR 结果（DescribeTaskStatus）
   直到 Status=2（成功）
  │
  ▼
5. 数据清洗（规则处理，不需要 AI）
   ├── 去除语气词：嗯、啊、呃、那个、就是、然后呢
   ├── 合并短句（时长 < 1秒的合并到下一句）
   ├── 拆分长句（超过 20字按标点断开）
   ├── 限制时长（每条 1~5秒）
   └── 限制行数（单条不超过 2行，每行不超过 20字）
  │
  ▼
6. 生成 SRT 字幕文件
   保存到：storage/output/xxx.srt
   同时保留原始识别结果：storage/output/xxx.raw.txt
  │
  ▼
7. FFmpeg 烧录字幕
   命令：ffmpeg -i input.mp4 -vf subtitles=storage/output/xxx.srt:force_style="..." storage/output/xxx.mp4
     （注：实际路径为 storage/input/ 和 storage/output/）
  │
  ▼
8. 清理临时文件（storage/temp/xxx.wav）
  │
  ▼
9. 返回结果：{ videoUrl, srtUrl, status: "done" }
```

## 字幕样式参数映射

| 前端字段 | SRT/ASS 参数 | FFmpeg force_style |
|---------|--------------|-------------------|
| 字体 | FontName | `FontName=Microsoft YaHei` |
| 字号 | FontSize | `FontSize=24` |
| 字体颜色 | PrimaryColour | `PrimaryColour=&H00FFFFFF`（BGR格式） |
| 描边颜色 | OutlineColour | `OutlineColour=&H00000000` |
| 描边粗细 | Outline | `Outline=2` |
| 位置（底部）| Alignment | `Alignment=2`（底部居中） |
| 位置（顶部）| Alignment | `Alignment=6`（顶部居中） |
| 距底部距离 | MarginV | `MarginV=30`（后端根据视频高度×百分比计算像素） |

### Alignment 对照表
```
7  8  9   （顶部：左 中 右）
4  5  6   （中部：左 中 右）
1  2  3   （底部：左 中 右）
```

## FFmpeg 命令示例

```bash
# marginV 换算公式：实际像素 = 视频高度 × marginV% / 100
# 例：视频高 1080px，marginV=10%  →  MarginV=108

# 提取音频（16kHz, 16bit, 单声道 WAV）
ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 output.wav

# 烧录字幕（带自定义样式）
ffmpeg -i input.mp4 \
  -vf "subtitles=output.srt:force_style='FontName=Microsoft YaHei,FontSize=24,PrimaryColour=&H00FFFFFF,OutlineColour=&H00000000,Outline=2,Shadow=1,Alignment=2,MarginV=30'" \
  -c:v libx264 -c:a aac output.mp4
```

## API 接口契约

### POST `/api/upload`

上传视频文件和字幕样式配置，开始处理任务。

**请求：** `multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| video | File | ✅ | 视频文件（MP4，30秒~3分钟） |
| style | String(JSON) | ✅ | 字幕样式配置（JSON字符串） |
| asrConfig | String(JSON) | ❌ | ASR 识别配置（可选，有默认值） |

**style JSON 结构：**
```json
{
  "fontName": "Microsoft YaHei",
  "fontSize": 24,
  "fontColor": "#FFFFFF",
  "outlineColor": "#000000",
  "outline": 2,
  "alignment": 2,
  "marginV": 10
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| fontName | String | "Microsoft YaHei" | 字体名称 |
| fontSize | Number | 24 | 字号（16~36） |
| fontColor | String | "#FFFFFF" | 字体颜色（HEX） |
| outlineColor | String | "#000000" | 描边颜色（HEX） |
| outline | Number | 2 | 描边粗细（0~5） |
| alignment | Number | 2 | 位置（2=底部 5=居中 8=顶部） |
| marginV | Number | 10 | 距边缘百分比（0~100），后端根据视频高度换算为实际像素 |

---

**asrConfig JSON 结构：**
```json
{
  "engineType": "16k_zh",
  "resTextFormat": 3
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| engineType | String | "16k_zh" | 识别引擎 |
| resTextFormat | Number | 3 | 结果格式（0=基础 1=+词时间戳 2=+标点 3=按标点分段） |

**engineType 可选值：**
| 值 | 说明 |
|------|------|
| 16k_zh | 中文普通话通用（默认） |
| 16k_zh_en | 普方英大模型 |
| 16k_en | 英语 |
| 16k_ja | 日语 |
| 16k_ko | 韩语 |
| 16k_yue | 粤语 |

**响应：**
```json
{
  "taskId": "abc123",
  "status": "uploading"
}
```

**错误响应：**
```json
{
  "error": "Invalid file format. Only MP4 is supported."
}
```

---

### GET `/api/status/:taskId`

查询任务处理状态。前端轮询此接口直到 status 为 `done` 或 `failed`。

**响应：**
```json
{
  "taskId": "abc123",
  "status": "recognizing",
  "progress": 60,
  "errorMsg": "",
  "videoUrl": null,
  "srtUrl": null,
  "rawUrl": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | String | 任务ID |
| status | String | `uploading` / `extracting` / `recognizing` / `cleaning` / `generating` / `rendering` / `done` / `failed` |
| progress | Number | 进度百分比（0~100） |
| errorMsg | String | 错误信息（仅 failed 时有值） |
| videoUrl | String | 带字幕视频下载路径（仅 done 时有值） |
| srtUrl | String | 字幕文件下载路径（仅 done 时有值） |
| rawUrl | String | 原始识别结果下载路径（仅 done 时有值） |

---

### GET `/api/download/:taskId/video`

下载带字幕的视频文件（仅 status=done 时可用）。

**响应：** 视频文件流（Content-Type: video/mp4）

---

### GET `/api/download/:taskId/srt`

下载字幕文件（仅 status=done 时可用）。

**响应：** SRT 文件流（Content-Type: text/plain; charset=utf-8）

---

### GET `/api/download/:taskId/raw`

下载原始识别结果（清洗前，仅 status=done 时可用）。

**响应：** 文本文件流（Content-Type: text/plain; charset=utf-8）

## 任务状态流转

> **前端第四步状态展示需包含 `cleaning`（数据清洗中...）**

```
uploading → extracting → recognizing → cleaning → generating → rendering → done
    ↑           ↑            ↑            ↑           ↑            ↑
  上传中     提取音频     语音识别中    数据清洗    生成字幕     渲染视频
                                                                     │
                                                                 failed（任意步骤失败）
```
