# srtGenerator 数据清洗与 SRT 生成

## 函数签名

```js
cleanAndGenerate(asrResult, baseName, log) → Promise<{ srtPath, rawPath }>
```

## 输入参数

| 参数 | 类型 | 说明 |
|------|------|------|
| asrResult | Array | ASR 返回的识别结果数组，每条：`{ StartTime, EndTime, Text }` |
| baseName | String | 文件名基础（不含扩展名），如 `"abc123"` |
| log | Object | Winston logger 实例，用于输出日志 |

### 输入数据格式

```js
// asrResult 每条的结构（processVideo.js 转换后的格式）
{
  StartTime: 0.5,      // 开始时间（秒）
  EndTime: 2.3,        // 结束时间（秒）
  Text: "大家好今天..." // 识别出的文字
}
```

## 输出

生成两个文件：

| 文件 | 说明 |
|------|------|
| `{baseName}.srt` | 清洗后的标准 SRT 字幕（给 FFmpeg 烧录用） |
| `{baseName}.raw.txt` | 原始识别结果备份（需求要求保留，方便对比） |

---

## 处理流程

```
ASR 原始结果
    │
    ▼
removeFillers    ← 去语气词（嗯、啊、呃...）
    │
    ▼
mergeShort       ← 合并过短句子（< 1 秒）
    │
    ▼
splitLong        ← 拆分过长句子（> 20 字）
    │
    ▼
mergeShort       ← 再合并一次（兜底拆分产生的残余）
    │
    ▼
过滤空文本       ← 去掉清洗后变成空的条目
    │
    ▼
限制时长         ← 上限 5 秒
    │
    ▼
validate         ← 验证 + 打警告日志（不改数据）
    │
    ▼
formatSrt        ← 转换为 SRT 格式字符串
    │
    ▼
写入文件
```

---

## 清洗规则详解

### 规则 1：removeFillers — 去除语气词

**为什么要去掉：** ASR 会忠实识别所有声音，包括"嗯""啊"，这些词出现在字幕里影响阅读体验。

**实现方式：**
```js
// 1. 构建正则：把语气词数组拼成 '嗯|啊|呃|...'
const pattern = new RegExp(FILLER_WORDS.join('|'), 'g')

// 2. 替换为空 + 清理空格
return text
  .replace(pattern, '')    // 删除语气词
  .replace(/\s+/g, ' ')    // 多个空格压成一个
  .trim()                  // 去掉首尾空格
```

**语气词列表：** `['嗯', '啊', '呃', '哦', '噢', '嗯嗯']`

---

### 规则 2：mergeShort — 合并过短句子

**为什么要合并：** 时长 < 1 秒的字幕观众来不及读，合并到相邻条目更好。

**合并逻辑：**

| 情况 | 处理方式 |
|------|----------|
| 中间某条 < 1 秒 | 文字拼到**下一条**开头，StartTime 改为当前的 StartTime |
| 最后一条 < 1 秒 | 文字追加到**上一条**末尾，EndTime 改为当前的 EndTime |

**示例：**
```
合并前：
  A: StartTime=0, EndTime=0.5, Text="嗯"
  B: StartTime=1, EndTime=5, Text="大家好"

合并后：
  B: StartTime=0, EndTime=5, Text="嗯大家好"  ← 时间范围向前扩展
```

---

### 规则 3：splitLong — 拆分过长句子

**为什么要拆分：** 一行字幕最多 20 字（适配手机小屏），超了要断成多条。

**拆分策略：**
1. 取前 20 字，从后往前找标点（，。！？；、）
2. 找到标点 → 在标点处断开
3. 找不到标点 → 在 20 字处硬切
4. 重复直到剩余 ≤ 20 字

**时间分配：** 按文字长度比例分配

```
原文 40 字，总时长 4 秒
拆成 20 字 + 20 字 → 每条 2 秒

原文 30 字，总时长 4 秒
拆成 20 字 + 10 字 → 第一条 2.67 秒，第二条 1.33 秒
```

---

### 规则 4：过滤空文本 + 限制时长

**过滤空文本：** 清洗后可能变成空字符串（比如原文全是语气词），直接丢掉。

**限制时长上限 5 秒：** 画面可能已经换了内容，字幕还在会造成困惑。

```js
segments = segments.filter(seg => seg.Text.trim().length > 0)
segments = segments.map(seg => ({
  ...seg,
  EndTime: seg.EndTime - seg.StartTime > 5 ? seg.StartTime + 5 : seg.EndTime
}))
```

---

## 验证函数

`validate(segments, log)` 只检查 + 打警告日志，**不修改数据**。

**检查项：**

| 检查项 | 阈值 | 警告内容 |
|--------|------|----------|
| 文字长度 | > 20 字 | `第 X 条超过 20 字：Y 字` |
| 时长下限 | < 1 秒 | `第 X 条时长不足 1 秒：Y 秒` |
| 时长上限 | > 5 秒 | `第 X 条时长超过 5 秒：Y 秒` |

---

## SRT 格式

每条字幕由 4 部分组成：

```
1                          ← 序号
00:00:01,500 --> 00:00:03,200   ← 时间范围（HH:MM:SS,mmm）
大家好，今天我们来聊一个话题    ← 字幕文字
                           ← 空行分隔
```

**时间格式注意：** SRT 用逗号 `,` 分隔毫秒，不是小数点 `.`

## 使用方式

```js
const { cleanAndGenerate } = require("./services/srtGenerator")

// asrResult 是从腾讯云 ASR 返回的 ResultDetail 转换后的数组
const asrResult = [
  { StartTime: 0.02, EndTime: 2.38, Text: "腾讯云语音识别欢迎您。" },
  // ...
]

const { srtPath, rawPath } = await cleanAndGenerate(asrResult, "abc123", log)
// srtPath: "storage/output/abc123.srt"
// rawPath: "storage/output/abc123.raw.txt"
```

## 设计决策

| 决策 | 原因 |
|------|------|
| 数据结构化优先 | 在对象上做业务逻辑，最后才转 SRT 格式 |
| 功能与验证分离 | 流水线只做功能操作，validate 只检查 + 日志 |
| 原始备份用 formatSrt | 格式统一，能直接用播放器打开对比 |
| mergeShort 跑两次 | 拆分可能产生过短尾部，需要兜底合并 |

---

## 性能优化方向

### 当前问题

srtGenerator 在**主线程**同步执行，而 FFmpeg/ASR 都是异步的。高并发时多个任务同时跑清洗逻辑会阻塞主线程，影响其他请求。

**当前遍历次数：** 约 6-7 次（removeFillers + mergeShort × 2 + splitLong + filter + map + validate）

### 优化方案：合并遍历

把多个 `.filter()` / `.map()` 合并成一次 `.reduce()`，减少遍历次数：

```js
// 优化前：6 次遍历
segments = segments.map(seg => ({ ...seg, Text: removeFillers(seg.Text) }))
segments = mergeShort(segments)
segments = splitLong(segments)
segments = mergeShort(segments)
segments = segments.filter(seg => seg.Text.trim().length > 0)
segments = segments.map(seg => ({ ...seg, EndTime: ... }))
segments = validate(segments, log)

// 优化后：把过滤 + 裁剪 + 验证合并成 1 次
segments = segments.reduce((acc, seg) => {
  // 1. 去语气词
  const text = removeFillers(seg.Text)
  if (text.trim().length === 0) return acc  // 过滤空文本

  // 2. 限制时长
  const duration = seg.EndTime - seg.StartTime
  const endTime = duration > 5 ? seg.StartTime + 5 : seg.EndTime

  // 3. 验证（打日志）
  if (text.length > 20) log.warn(...)
  if (endTime - seg.StartTime < 1) log.warn(...)

  acc.push({ ...seg, Text: text, EndTime: endTime })
  return acc
}, [])
```

**效果：** 6-7 次遍历 → 1 次遍历，性能提升明显。

### 未来方向：队列限流

如果并发量真的很大，用队列限制同时处理的任务数：

```js
const PQueue = require('p-queue')
const queue = new PQueue({ concurrency: 3 })  // 最多 3 个任务同时处理

app.post('/upload', (req, res) => {
  queue.add(() => processVideo(taskId))  // 超过 3 个的任务排队
  res.json({ taskId })
})
```

**效果：** 即使清洗逻辑没优化，也不会同时阻塞主线程。

### 未来方向：Worker Threads 多线程

把 CPU 密集型的清洗逻辑放到独立线程，完全不阻塞主线程：

```js
// worker.js — 在独立线程运行
const { parentPort } = require('worker_threads')
const { cleanAndGenerate } = require('./srtGenerator')

parentPort.on('message', async (data) => {
  const result = await cleanAndGenerate(data.asrResult, data.baseName, data.logPath)
  parentPort.postMessage(result)
})

// 主线程调用
const { Worker } = require('worker_threads')

function runInWorker(asrResult, baseName) {
  return new Promise((resolve, reject) => {
    const worker = new Worker('./worker.js')
    worker.postMessage({ asrResult, baseName })
    worker.on('message', resolve)
    worker.on('error', reject)
  })
}

// processVideo.js 里
await runInWorker(asrResult, baseName)
```

**效果：** 清洗逻辑在后台线程跑，主线程继续处理其他请求，零阻塞。

**注意：** Worker 线程无法共享主线程的 logger 对象，需要传 logPath 让 Worker 自己创建 logger。

### 优化方向：省去格式转换遍历

当前 processVideo 在调用 cleanAndGenerate 前有一次格式转换遍历（ASR 格式 → 内部格式）。可以让 srtGenerator 直接接收 ASR 格式（FinalSentence, StartMs, EndMs），内部处理毫秒单位，省去这次遍历。
