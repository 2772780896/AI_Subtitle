# route.js 代码说明

> 本文档记录 `backend/routes/route.js` 的处理流程。
> route.js 是 HTTP 薄层，只负责接收请求和返回响应，不含业务逻辑。

---

## 5 个 API

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/upload` | POST | 接收视频 + 样式配置，创建任务，启动处理 |
| `/api/status/:taskId` | GET | 查询任务状态（前端轮询） |
| `/api/download/:taskId/video` | GET | 下载带字幕视频 |
| `/api/download/:taskId/srt` | GET | 下载字幕文件 |
| `/api/download/:taskId/raw` | GET | 下载原始识别结果 |

---

## POST /upload 流程

```
前端 POST FormData (video + style + asrConfig)
  │
  ▼
① upload.single('video') 中间件（multer）
   - 文件保存到 storage/input/，UUID 重命名
   - 文件信息 → req.file
   - 其他字段 → req.body
  │
  ▼
② 文件校验：req.file 存在？后缀是 .mp4？
   - 非 MP4 → fs.unlinkSync 删掉 → 返回 400
  │
  ▼
③ 解析 style（JSON.parse）和 asrConfig（有默认值，用户值覆盖）
  │
  ▼
④ 创建任务：生成 taskId（UUID 前 8 位），存入 Map
   - 公开字段：taskId / status / progress / errorMsg / videoUrl / srtUrl / rawUrl
   - 内部字段（_前缀）：_videoPath / _videoName / _style / _asrConfig
  │
  ▼
⑤ 启动 processVideo(taskId)（故意不 await，后台异步处理）
  │
  ▼
⑥ 立即返回 { taskId, status: 'uploading' }
```

**关键设计：** processVideo 是 async 函数，但 route.js 故意不 await 它。
这样 HTTP 响应立即返回，processVideo 在后台继续跑，前端通过轮询 `/api/status` 获取进度。

---

## GET /status/:taskId 流程

```
取 taskId → Map 中查找 → 不存在返回 404
  │
  ▼
返回公开字段（不返回 _前缀的内部字段）
```

前端每 2 秒轮询一次，根据 status 更新进度条。done 或 failed 时停止轮询。

---

## GET /download/:taskId/video、/srt、/raw 流程

三个下载路由逻辑相同：

```
取 taskId → Map 中查找 → 不存在返回 404
  │
  ▼
检查 task.status === 'done' → 未完成返回 400
  │
  ▼
根据 _videoName 计算文件路径 → fs.existsSync 检查 → 不存在返回 404
  │
  ▼
res.download(filePath) 强制下载
```

| 路由 | 文件路径 | 文件名规则 |
|------|----------|------------|
| /video | `storage/output/{_videoName}` | 原始 UUID 名（如 `abc.mp4`） |
| /srt | `storage/output/{_videoName}.srt` | 把 `.mp4` 换成 `.srt` |
| /raw | `storage/output/{_videoName}.raw.txt` | 把 `.mp4` 换成 `.raw.txt` |

## multer 配置

```js
const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    cb(null, path.join(__dirname, '../storage/input'))  // 存到 input/
  },
  filename: (req, file, cb) => {
    const ext = path.extname(file.originalname)
    cb(null, `${uuidv4()}${ext}`)  // UUID 重命名，防止文件名冲突
  }
})
```
