# route.js 代码说明

> 本文档记录 `backend/routes/route.js` 的处理流程。遇到不懂的概念时再补充说明。

---

## 4 个 API

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/upload` | POST | 接收视频 + 样式配置，创建任务 |
| `/api/status/:taskId` | GET | 查询任务状态（前端轮询） |
| `/api/download/:taskId/video` | GET | 下载带字幕视频 |
| `/api/download/:taskId/srt` | GET | 下载字幕文件 |

---

## POST /upload 流程

```
前端 POST FormData (video + style + asrConfig)
  │
  ▼
① upload.single('video') 中间件
   - 文件保存到 input/，UUID 重命名
   - 文件信息 → req.file
   - 其他字段 → req.body
  │
  ▼
② 文件校验：req.file 存在？后缀是 .mp4？
   - 非 MP4 → fs.unlinkSync 删掉 → 返回 400
  │
  ▼
③ 解析 style（JSON.parse）和 asrConfig（有默认值，用户值覆盖）
  │
  ▼
④ 创建任务：生成 taskId，存入 Map
   - 公开字段：taskId / status / progress / videoUrl / srtUrl
   - 内部字段（_前缀）：_videoPath / _videoName / _style / _asrConfig
  │
  ▼
⑤ 启动 mockProcess 模拟状态流转（后续换成真实处理）
  │
  ▼
⑥ 返回 { taskId, status: 'uploading' }
```

---

## GET /status/:taskId 流程

```
取 taskId → Map 中查找 → 不存在返回 404
  │
  ▼
返回公开字段（不返回 _前缀的内部字段）
```

前端每 3 秒轮询一次，根据 status 更新进度条。

---

## GET /download/:taskId/video 和 /srt 流程

```
取 taskId → Map 中查找 → 不存在返回 404
  │
  ▼
检查 task.status === 'done' → 未完成返回 400
  │
  ▼
检查文件是否存在（fs.existsSync）→ 不存在返回 404
  │
  ▼
res.download(filePath) 强制下载
```

---

## mockProcess 模拟流转

用 `setTimeout` + 累加延迟，模拟 7 个阶段（约 7 秒）：

```
uploading(10%) → extracting(30%) → recognizing(50%) → cleaning(70%) → generating(85%) → rendering(95%) → done(100%)
```

done 时设置 videoUrl 和 srtUrl 为 `/api/download/${taskId}/...`。
